# Guía de Contribución - Eventos Chiapas

Para mantener un historial limpio y organizado en este proyecto, el equipo siguió las siguientes reglas de trabajo:

## Reglas para Ramas
Cada integrante debió trabajar en una rama independiente creada a partir de `main` utilizando la estructura:
* `rama-[nombre-seccion]` (Ejemplo: `rama-header`, `rama-contacto`, `rama-chiapas`)

## Mensajes de Commit
Los commits debieron ser frecuentes (mínimo 3 por rama) y con mensajes claros en tiempo presente que describieran la acción exacta realizada:
* Ejemplo: `"se agrego formulario de contacto"`
* Ejemplo: `"se mejoro diseño de la pagina de eventos"`

## Proceso de Fusión
Ningún cambio se subió directamente a `main`. Se requirió la apertura de un Pull Request (PR) y la revisión/aprobación de otro compañero del equipo antes de realizar el Merge.
